Intuitive Syntax

Ema uses intuitive syntax that was developed for compatibility with state-of-the art research tools like OpenSees, FERUM, and FEDEASLab. ema-compiled numerical algorithms in Python can approach the speeds of C or FORTRAN.

You don't need to replace the Python interpreter, run a separate compilation step, or even have a C/C++ compiler installed. Just apply one of the ema decorators to your Python function, and ema does the rest.

Built for Scientific Computing

ema is designed to be used with NumPy arrays and functions. ema generates specialized code for different array data types and layouts to optimize performance. Special decorators can create universal functions that broadcast over NumPy arrays just like NumPy functions do.

ema also works great with Jupyter notebooks for interactive computing, and with distributed execution frameworks, like Dask and Spark.

    def logistic_regression(Y, X, w, iterations):

            w -= np.dot(((1.0 /
                  (1.0 + np.exp(-Y * np.dot(X, w)))
                  - 1.0) * Y), X)
        return w

Portable Data Structure

Ship high performance Python applications without the headache of binary compilation and packaging. Your source code remains pure Python while ema handles the compilation at runtime. We test ema continuously in more than 200 different platform configurations.

ema supports Intel and AMD x86, POWER8/9, and ARM CPUs, NVIDIA and AMD GPUs, Python 2.7 and 3.4-3.7, as well as Windows/macOS/Linux. Precompiled ema binaries for most systems are available as conda packages and pip-installable wheels.
